pingpong

pipe能够创建两个fd，0对应input，使用read获取；1对应output，使用write输出

这里两个进程不是使用一个管道通信，而是两个管道，每个pipe都绑定了一个进程，这个进程可以对pipe进行write和read

使用两个pipe可以进行进程之间的互相通信，一个不行。

primes

使用管道实现素数筛法

首先在main函数里面，即主进程，我们需要把所有数字输入

然后我们调用写好的函数来完成素数筛，传入的是主进程的input

在solve中，使用read先读取第一个数字，作为起点，然后读取input流里面所有的数字并筛，如果一个数没有被筛掉，那么传递给下一个进程处理

重复上述过程，直到读完input中的数字，然后我们创建子进程进行进一步处理

在创建下一个进程的时候，我们需要等在子进程指向完毕才释放，所以使用wait来等待子进程完成

find

找到具有tar模式的文件名，输出它的路径

提示：

  • Look at user/ls.c to see how to read directories.
  • Use recursion to allow find to descend into sub-directories.
  • Don’t recurse into “.” and “..”.
  • Changes to the file system persist across runs of qemu; to get a clean file system run make clean and then make qemu.
  • You’ll need to use C strings. Have a look at K&R (the C book), for example Section 5.5.
  • Note that == does not compare strings like in Python. Use strcmp() instead.
  • Add the program to UPROGS in Makefile.

fmtname 用于获取文件名

open 用于打开文件，返回文件描述符fd（目录也是一个文件）

fstat(fd, &st) 用于获取文件信息到info中

然后根据st的type进行判断，如果是文件，我们判断文件名是否相同

strcpy(buf, path)是把path复制到buf中

memmove(p, de.name, DIRSIZE) 是将de.name 中的 DIRSIZE 复制到p中

思路：

首先判断dst是文件还是目录

总之我们先尝试打开dst

然后判断dst类型

如果dst是文件，我们直接比较名称是否相等

如果dst是目录，那么需要遍历其目录下除了.和..以外的所有文件，递归处理，稍微有点恶心的地方就是文件名了
