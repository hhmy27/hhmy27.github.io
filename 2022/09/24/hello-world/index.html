配置Github Actions

观察hexo的目录结构可以发现

public文件夹里面的内容，就是我们{username}.github.io仓库里面的内容

而source文件夹是我们.md文件存放的位置

实际推送的时候，只是把public文件夹里面的内容推送到远程的{username}.github.io仓库

为了避免数据丢失，我们自然而然的想到备份hexo仓库

我查询了一下资料，常用的解决方法有两种：

  1. 在github.io仓库里面，添加一个新的分支，存放hexo的内容

    思考了一下，这样做可能会涉及一些安全问题，而且管理也不是那么清爽

  2. 新建一个private仓库，使用该仓库存放hexo的内容，使用 hexo-git-backup 进行备份，然后使用 Github Actions 添加工作流，自动更新 github.io仓库

    本博客就是使用的该方案配置

配置GitHub Actions方法可以参考 Github Actions 初体验之自动化部署 Hexo 博客 (razeen.me)，这里不赘述了

最终实现效果是当我们使用hexo b的时候，会备份hexo配置仓库，然后自动更新github.io仓库的内容

完成效果

最终有两个 github 仓库，一个是 hexo_backup，用来保存源文件和配置，另一个是公开的 github.io 博客
